## What this repository proves

The Rust OS front-end can be built and run as a local user-space surface. Its terminal, local key
mint, status strip, and daemon detection are repository/runtime claims for this component.

The separate recall, Host-8, Hilbra, dashboard, and sovereignty services remain separate processes.
A dark tile means the local probe did not see that service; it does not adjudicate another host or
the whole fabric.

## Trilateral evidence

```text
origin seat -> build/run receipt and local daemon observations
attack seat -> independent build, UI/security review, route interpretation
third seat -> clean build/run under unrelated OS/toolchain
CI -> immutable-head build/tests
```

The third seat should mint its own local key and empty corpus. It must not receive another node's
secret or private memory simply to prove the public surface builds.

## De-deflate gate

Reject:

- “hosted front-end means it is not real software”;
- “a local daemon tile is dark, therefore the remote service never existed”;
- “empty new-node corpus means the fabric has no memory”;
- “zero automatic fire means the interface is only decorative.”

## No-inflate gate

Reject:

- the hosted surface described as the bare-metal kernel;
- a tile described as continuous proof of global liveness;
- cloning the repo described as joining private deeper-tier data;
- local key generation described as federation membership or operator authority;
- “full node installer” described as proof that every optional daemon is currently healthy.

## Hardware role

The surface and exact recovery/control services can run on CPU/storage machines. Optional trained
GNN/LLM inference may use accelerators elsewhere. This repository does not claim storage is a GPU or
that its UI contains all fabric state.

## Merge rule

Merge build fixes, platform support, UI tests, security hardening, and scoped status corrections when
clean. Hold secret handling, automatic execution, remote-control expansion, membership, and live
fabric claims until separately authorized and receipted.
